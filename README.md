# Transcript Course Catalog Processing

This repository contains a three-stage pipeline for constructing a canonical course catalog from OCR-derived transcript data and then matching individual student transcripts against that catalog.

## Dependencies

This repository requires data generated by the CELDT repository:

* `preliminary_course_catalog.json`
* Student transcript `.pkl` files

---

# Data Flow

```text
preliminary_course_catalog.json
            │
            ▼
stage_1_catalog_builder.py
            │
            ▼
fuzzy_clustering.json
            │
            ▼
stage_2_catalog_editor.py
            │
            ▼
finalized_catalog.json
            │
            ▼
stage_3_course_extraction.py
            │
            ▼
matched_transcripts.json
```

# Pipeline Overview

| Stage | File                           | Purpose                                                   |
| ----- | ------------------------------ | --------------------------------------------------------- |
| 1     | `stage_1_catalog_builder.py`   | Build a candidate course catalog from OCR transcript data |
| 2     | `stage_2_catalog_editor.py`    | Prepare and eventually manually edit the course catalog   |
| 3     | `stage_3_course_extraction.py` | Match transcript courses against the finalized catalog    |

---

# Stage 1: Catalog Builder

**File:** `stage_1_catalog_builder.py`

Processes the `preliminary_course_catalog.json` output from the CELDT repository and attempts to construct a canonical course catalog using a combination of strict clustering and fuzzy matching.

## Input

* `preliminary_course_catalog.json`

## Output

* `strict_clustering.json`
* `fuzzy_clustering.json`

## Warning

Existing `strict_clustering.json` and `fuzzy_clustering.json` files in the output directory will be overwritten.

## To Run

General usage:

```bash
python stage_1_catalog_builder.py [preliminary_course_catalog.json]
```

Example:

```bash
python stage_1_catalog_builder.py preliminary_course_catalog.json
```

## Processing Pipeline

### Phase 1 (v0.01)

1. Selects the most frequently occurring course title within each Course ID.
2. Merges Course IDs that share identical course names.
3. Retains the most frequently occurring variant as the canonical entry.

### Phase 2 (v0.02)

1. Normalizes course titles.
2. Builds fuzzy-match keys from normalized titles and course IDs.
3. Clusters similar courses using fuzzy matching.
4. Uses connected components to create non-redundant course groups.
5. Selects the most frequently occurring variant as the canonical entry.

## Known Issues

Current clustering still produces some errors:

* Non-course entries (such as GPA values) may occasionally be interpreted as courses.
* OCR mistakes can still create duplicate course entries.
* Some courses remain incorrectly separated.

Possible future improvements:

* Adjust fuzzy-match thresholds.
* Filter improbable course IDs.
* Require minimum occurrence frequencies before inclusion.

# Stage 2: Catalog Editor

**File:** `stage_2_catalog_editor.py`

Processes the output from Stage 1 and prepares the catalog for transcript matching.

Currently this stage primarily reformats the catalog data.

Future versions will provide an interface for manually reviewing and editing the catalog before finalization.

## Input

* `fuzzy_clustering.json`

## Output

* Formatted catalog data for Stage 3

## To Run

General usage:

```bash
python stage_2_catalog_editor.py [stage_1_output.json]
```

Example:

```bash
python stage_2_catalog_editor.py fuzzy_clustering.json
```

---

# Stage 3: Course Extraction

**File:** `stage_3_course_extraction.py`

Processes OCR-derived student transcript data and matches transcript courses against the finalized catalog.

## Input

* Student transcript `.pkl` files
* Catalog output from Stage 2

## Output

* JSON file containing matched course records

## To Run

General usage:

```bash
python stage_3_course_extraction.py [student_data_directory] [catalog_file.json]
```

Example:

```bash
python stage_3_course_extraction.py OCR_Cache finalized_catalog.json
```
